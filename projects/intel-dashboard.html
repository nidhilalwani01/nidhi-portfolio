Overview

This project simulates live event check-in operations for the Intel Sustainability Summit. The dashboard is designed to help organizers quickly track attendance status, monitor team performance, and respond to operational changes with clear, data-driven visibility.

Problem

During high-volume events, check-in data can become difficult to interpret in real time. Teams need a concise operational view that surfaces who is checked in, where bottlenecks are forming, and how progress compares across teams.

Goals

  • Provide clear attendance tracking at a glance.
  • Enable team-based analytics for quick performance comparison.
  • Visualize event progress dynamically for rapid status checks.
  • Support fast operational decisions during live event conditions.

Dashboard Design Approach

  • Information Hierarchy: Prioritized high-impact metrics in the first view.
  • Task-Oriented Layout: Grouped attendance, team analytics, and progress modules by decision flow.
  • Operational Clarity: Used concise labels and status cues to reduce scan time.
  • Consistency: Applied repeatable card patterns for easier interpretation under pressure.

Interaction and Data Visualization Decisions

  • Used dynamic progress visuals to show live completion status.
  • Designed clear states for checked-in, pending, and delayed attendees.
  • Highlighted team-level comparisons to expose operational imbalance quickly.
  • Kept interaction patterns lightweight to maintain performance and responsiveness.

Implementation Highlights

  • Built with semantic HTML structure for maintainability.
  • Styled using modular CSS sections for reusable dashboard components.
  • Used JavaScript to update live counters and progress indicators.
  • Optimized layout behavior for desktop and tablet event operations.

Final Outcome

The completed dashboard demonstrates a practical model for live event monitoring. It combines attendance visibility, team performance insights, and real-time progress signals into a single operational interface that supports faster and more confident decision-making.

Reflections

  • Dashboard value depends on prioritizing the right metrics at first glance.
  • Simple visual states can reduce cognitive load during live operations.
  • Small interaction details improve monitoring speed and confidence.
  • A next iteration could include role-based views for different operations teams.
